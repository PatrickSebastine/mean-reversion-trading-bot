## How to contribute

Contributions are welcome. To propose a change:

1. Open an issue describing the bug, feature, or improvement.
2. Fork the repository and create a branch for your change.
3. Keep changes focused and match the existing style.
4. Never commit secrets, API keys, or `.env` files. Use `.env.example` as the template.
5. Open a pull request that explains what changed and why.

Good first areas to help with: new strategy modules, additional indicators, exchange-specific fixes, and backtest reporting.
